### ⚠️ OBSERVACIONES CRÍTICAS

#### 1. **CRÍTICA**: Schema Flyway Desactualizado
**Archivo:** `V1__Initial_Schema.sql`  
**Problema:** El schema no coincide con JobJpaEntity

| Campo en JobJpaEntity | Tipo Java | Columna en DB | Tipo SQL | Estado |
|---|---|---|---|---|
| `universalId` | String | `universal_id` | UUID | ❌ MISMATCH |
| `title` | String | `title` | VARCHAR(200) | ✅ OK |
| `description` | String | `description` | TEXT | ✅ OK |
| `version` | Long | `version` | ??? | ❌ FALTA |
| `offeredBy` | String | `offered_by` | ??? | ❌ FALTA |
| `companyName` | String | `company_name` | ??? | ❌ FALTA |

**Impacto:** 
- Compilación fallará al intentar mapear `String universalId` a `UUID` en clase
- Campo `version` NO existe en DB → Optimistic locking no funcionará
- Campos `offeredBy` y `companyName` NO existen en schema

#### 2. **CRÍTICA**: Columnas Faltantes en Schema
El script Flyway define:
- ❌ NO hay `version` (necesario para @Version)
- ❌ NO hay `offered_by` 
- ❌ NO hay `company_name`
- ❌ NO hay `industry_id`
- ❌ NO hay `region_id`

Pero JobJpaEntity espera estos campos.

#### 3. **CRÍTICA**: Mapeo de Location
En schema: `location_address1`, `location_address2`, etc. (14+ columnas denormalizadas)  
En JobJpaEntity: `@Embedded private JobLocationEmbeddable location;`

**Problema:** Los nombres de columnas NO coinciden. JobJpaEntity usa @AttributeOverrides o mapeos automáticos por nombre, pero los nombres en schema son diferentes.

#### 4. **CRÍTICA**: Mapeo de Salary
En schema: `salary_min`, `salary_max`, `salary_currency`  
En JobJpaEntity: `@Embedded private JobSalaryEmbeddable salary;`

**Problema:** Misma situación que Location — nombres no coinciden.
